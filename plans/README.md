### Findings considerados e rejeitados

- Substituir `NSEvent` local key monitors por `FocusState` + comandos nativos
  SwiftUI: rejeitado porque os key monitors têm lógica de scoping complexa que
  `FocusState` não cobre (precisa saber se command palette está aberto, se
  text input está focado, se popover está visível).
- Adicionar `DisclosureGroup` para seções colapsáveis: rejeitado porque a
  implementação atual com `@AppStorage` para persistência não é compatível com
  `DisclosureGroup`.
- Full redesign do sistema de navegação (NavigationSplitView): rejeitado porque
  a estrutura de popover/sheet/window é adequada para menu bar app.

## Findings considered and rejected (original)

- Full `GitManager.swift` decomposition as one big-bang refactor: rejected in favor of
  incremental slices behind a stable facade; first slice is **Plan 007** (GitBranchService),
  with GitAtomicCommitService / GitCommitHistoryService as deferred follow-ups.
- Merge conflict resolver UI: too complex for this round; conflicts delegate to manual resolution
- Git LFS support: out of scope
- Merge `menubar` into the new macOS UI skill: rejected because status-item ownership,
  click behavior, and popover dismissal are product-critical invariants for GitMenuBar.
- Merge `test-strategy` into delivery guidance: rejected because delivery workflow owns
  gates and command routing, while `test-strategy` owns test design, fakes, async coverage,
  and regression cases.
- Add a general debugging skill in the same pass: rejected because the current overlap is
  already solved by delivery/macOS consolidation; debugging can be planned separately if
  repeated investigations show a routing problem.
- Require tests before every commit: rejected because the repo has no active
  hooks, `make test` already performs `build-for-testing`, and mandatory full
  tests on every commit would slow small agent iterations while duplicating the
  merge/push gate.
- Keep the current `make lint-changed` as-is: rejected because the current
  target misses untracked Swift files and falls back to full lint when no
  changed tracked Swift files exist.
